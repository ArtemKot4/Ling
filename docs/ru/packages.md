# Пакеты ☑
Пакеты созданы для того, чтобы упаковывать функции и переводы в некие модули, к которым мы сможем получить доступ по названию.

Это нужно для того, чтобы структурировать данные, например математические функции будут лежать в пакете `math`, а описание сообщений для пользователя в `user.messages`

Пакеты складываются. Вы можете использовать одни и те же пакеты из разных файлов, не теряя доступа к ключам внутри.

Пакеты не могут быть пустыми, в них обязаны храниться функции или переводы.

## Создание
Для того, чтобы создать пакет, вам нужно использовать ключевое слово `package`.

Вы можете дать пакету любое название, пока он не конфликтует с ключевыми словами и состоит из английского алфавита и цифр или точек, нижних подчёркиваний. В другом случае вы можете поместить название в строку.

Создадим пакет для хранения некоторых сообщений для пользователя на главной странице:
```ling
define langs = en-US, ru-RU //определили языки

package home {
    hi = "Hi", "Привет" //создали перевод под наши языки в нашем пакете
    /**
     * Функция, возвращающая текст в формате `Hi, %name!`
     */
    get_hi(name) { //создали функцию в нашем пакете
        "${hi}, ${name}!"
    }
}

package "with_string" {
    type = "String", "Строка"
}
```

## Получение значений
Ранее мы определили пакет `home`. Давайте создадим пакет `implementation` и применим данные из первого пакета.

```ling
package implementation {
    hi_jack = home.hi + ", Jack!" //Hi, Jack! Привет, Jack!

    hi_ann = home.get_hi("Ann") //Hi, Ann! Привет, Ann!
}
```

## Вложенные пакеты
Пакеты могут бесконечно вкладываться друг в друга.
```ling
define langs = ru-RU

package first {
    package second {
        package third {
            value = "Значение"
        }
    }
}

get_value() {
    first.second.third.value //получили значение из последнего вложенного пакета
}
```
Так же вы можете использовать сокращённую запись, и сделать так:

```ling
define langs = ru-RU

package first.second.third.value {
    value = "Значение"
}
```

## Области видимости
Вы можете использовать значения по прямому названию, пока они лежат в том же пакете, в котором вы их применяете. В другом случае вам нужно будет обращаться к ним по относительному пути пакета.

Все пакеты изначально наследуют `langs` и `unexpected` из последнего объявленного в основном коде define, но вы можете переопределить параметры внутри пакета. 

В таком случае все вложенные паки будут наследовать именно ваш определённый внутри пакета define.

```ling
define langs = en-US

package hello {
    hello = "hello"
    package howdy {
        define langs = en-US, ru-RU
        howdy = "Howdy!", "Привет!"
    }
    lol = "lol"
}

```
