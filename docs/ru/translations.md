# Переводы ☑

Хранение переводов - одна из основных задач языка Ling.

Для их создания достаточно написать название ключа, поставить знак равно и перечислить переводы через запятую.

В случае, если вы написали лишние переводы, вы получите ошибку

В случае, если вы написали недостаточное количество переводов и не использовали [оператор нулевого слияния переводов](#оператор-нулевого-слияния-переводов), вы получите ошибку с перечислением языков, для которых переводов не хватило.

## Создание

Давайте создадим простой перевод:

```ling
define langs = en-US, ru-RU
hello = "Hello!", "Здравствуйте!"
```

## Использование выражений

Вы можете создавать переводы с использованием различных математических выражений:

```ling
define langs = en-US
ten = "5 + 5 is " + (5 + 5)
```

Все они вычислятся на этапе компиляции, превратившись в итоговые строки.

## Оператор нулевого слияния переводов

Оператор `?` позволяет указать транспилятору, что вместо ошибки нужно подставлять значение null в значения локализации перевода, которые не указаны.

При его использовании вы получите предупреждение в консоли: 

```txt
Nullable values for langs "перечисление_языков" by translation "название_ключа_перевода" at package "название_пакета" not recommended
```

Этот оператор создан скорее для быстрой разработки, в случае если вы не успеваете доделать все необходимые переводы. 

После публикации или обновления проекта лучше его не использовать, если это поведение не является запланированным.

Например:

```ling
define langs = en-US, ru-RU

package home {
    main = "Main room" //Ошибка: ожидался перевод для ru-RU
    //или
    main? = "Main room" //Значение для ru-RU стало null
    //или
    main = "Main room", "Главная комната" //Идеальный вариант, всех переводов хватает, ошибок и null значений нет
}
```
