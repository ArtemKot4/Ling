# Ling

Ling - прогрессивный предметно-ориентированный декларативный язык, созданный на базе TypeScript для создания и управления локализацией и форматированием текста в более удобном формате.

### В чём проблема современных решений?

Современные способы хранения данных, такие как json и yaml работают отлично, но хранят данные они статично. Это создаёт проблему того, что переводы мы описываем на одной стороне, а их форматирование на другой, что не очень удобно и не всегда легко читать. 

## Почему Ling?

Ling предлагает не только описывать переводы, но и функции их обработки, что позволяет не только сделать код визуально проще, но и сразу же видеть в файлах локализации, по какой логике текст обрабатывается.



**Все основные особенности**:

* Визуальное помещение данных в пакеты, что проще читать

* Возможность при изменении пакетов получать и изменения в связанных выражениях

* Обработка ошибок, в том числе с попыткой получить несуществующий ключ (`unexpected`) и с не полностью раскрытыми переводами - например вы создали файл с несколькими языками, но забыли написать перевод для какого-то из них

* Способность форматировать текст при помощи функций не прибегая к js

* Простой синтаксис и низкий порог входа


Давайте напишем простой пример функции в Ling:

```ling
define langs = en-US, ru-RU

package word {
    not = "not", "не"
    palindrome = "palindrome", "палиндром"

    is_palindrome(word) {
        match word == word[..-1]: word + " " + palindrome
        word + not + " " + palindrome
    }
}
```

Эта функция позволяет вывести строку, которая сообщит, является ли переданное слово палиндромом. 

Это был лишь один пример, а теперь представьте, что будет, если мы сможем в таком простом формате создавать склонения, сокращения и не только, не прибегая к js. В этом и заключается смысл функций в Ling.

### Что насчёт скорости?

Поскольку язык хоть и является декларативным, но может содержать в себе выражения, они вычисляются в момент получения перевода в случае, если они там есть (а это вызов функций, математические операции с данными из пакетов и так далее). Это может быть не самым быстрым процессом, но и не самым медленным.

Однако для проектов, в которых скорость имеет очень большую роль (чаще всего это проекты, где переводы не отдаются сервером, а загружаются на клиенте), будет разработана генерация js файлов, где будут лежать переводы и готовые js функции.

### Как использовать?

В данный момент ещё не вышла первая стабильная версия, ведь язык находится в активной разработке. Реализованы пакеты, определения и некоторые другие вещи, о которых вы можете узнать из [спецификации](./SPECIFICATION-ru.md).

После релиза, для использования будет достаточно скомпилировать файлы при помощи встроенных инструментов, а для подсветки и подсказок использовать специальный плагин.
